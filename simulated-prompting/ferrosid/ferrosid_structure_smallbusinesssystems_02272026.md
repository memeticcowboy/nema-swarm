## STEP 3: RESEARCH FINDINGS

### Framework 1: 5S Visual Management (Lean Manufacturing)

**Core Principles:**
- **Sort** - Remove unnecessary items from workspace
- **Set in Order** - Organize remaining items for easy access
- **Shine** - Clean and inspect the workspace
- **Standardize** - Create consistent practices
- **Sustain** - Maintain discipline through habit

**Lightweight Implementation for Marcus:**
- Color-code green coffee storage by origin/roast profile
- Shadow boards for tools (scoop, thermometer, scale)
- Visual "home" indicators on shelves for inventory
- 10-minute end-of-day cleanup ritual

**Warning Signs:**
- If 5S becomes about perfection rather than function
- If team sees it as "extra work" rather than "making work easier"
- If Marcus is the only one enforcing it

**Estimated Effort:** 2-3 hours initial setup, 10 minutes daily maintenance

---

### Framework 2: Two-Bin Kanban Inventory System

**Core Principles:**
- Visual pull-based replenishment
- No complex software required
- Prevents stockouts without overstocking
- Reduces decision fatigue

**Lightweight Implementation for Marcus:**
- Two bins per green coffee origin
- When Bin 1 empties, move to Bin 2 and reorder
- Simple card system: "REORDER" placed when bin is empty
- Par levels set based on 2-week usage + safety buffer

**Warning Signs:**
- If bins are too large (hides problems)
- If reorder quantities are wrong (leads to overstock)
- If system isn't physically obvious (defeats visual purpose)

**Estimated Effort:** 4 hours initial setup, 5 minutes daily

---

### Framework 3: Standard Work (Not Standard Procedures)

**Core Principles:**
- Documents current best practice, not ideal state
- Living document owned by operators
- Focus on key points, not every step
- One-page maximum

**Lightweight Implementation for Marcus:**
- Roast profile cards: origin, temp curve, key checkpoints
- Bagging station checklist: 5 items max
- Cleaning checklist with photos, not words
- Weekly team review: "What should change?"

**Warning Signs:**
- If documents live in a binder nobody opens
- If they're written by Marcus alone
- If they describe "how it should be" not "how it is"

**Estimated Effort:** 1 hour per process, updated monthly in 15-minute huddles

---

### Framework 4: Rhythm-Based Scheduling (Takt Time Lite)

**Core Principles:**
- Regular cadence beats reactive firefighting
- Small batches on consistent schedule
- Predictability enables delegation

**Lightweight Implementation for Marcus:**
- Fixed roasting days: Mon/Wed/Fri mornings
- Fixed bagging/packing days: Tue/Thu
- Fixed admin day: Friday afternoon
- Flex slot: Thursday for overflow/rush orders

**Warning Signs:**
- If schedule is too rigid (can't handle real demand)
- If Marcus is the only one who knows the schedule
- If "exceptions" become the rule

**Estimated Effort:** 2 hours planning, posted on wall, reviewed weekly

---

### Framework 5: Ownership Zones (RACI for Tiny Teams)

**Core Principles:**
- One person owns each zone (not does all the work)
- Authority to make decisions in their zone
- Cross-training reduces single points of failure
- Marcus moves from "doer" to "coach"

**Lightweight Implementation for Marcus:**
- Green Coffee Inventory: Employee A owns
- Roast Operations: Employee B owns
- Packaging/Shipping: Employee C owns
- Quality/Testing: Marcus owns (for now)
- Weekly 15-min handoff meeting between zone owners

**Warning Signs:**
- If zones become silos (no communication)
- If Marcus can't let go of decisions
- If ownership isn't matched with authority

**Estimated Effort:** 1 hour conversation per employee, 15 minutes weekly

## STEP 4: COWBOY NOTES

**FERROSID Analysis (Metal/Structure Lens):**

Marcus is experiencing classic **Structural Inflation**—he believes "without this form, everything falls apart." The truth is more subtle: the business has outgrown his informal coordination style, but he fears formal structure will kill what makes it special.

**G-Verb Application:**

- **Gates:** Marcus needs to define what ONLY he decides vs. what others can decide. Currently, everything flows through him—this is the bottleneck.

- **Grids:** The business needs repeating patterns (roasting days, inventory checks, team huddles). Chaos comes from everything being reactive.

- **Guards:** Two-bin inventory prevents stockouts. Standard Work prevents quality drift. These are minimum viable protections.

- **Guides:** Documentation should answer "what do I do when..." not "here is the complete theory of coffee roasting."

- **Gauges:** Three metrics max: inventory turns, on-time delivery, team hours. Everything else is noise at this stage.

- **Grounds:** The non-negotiable foundation is that systems serve the craft, not the other way around.

**Failure Mode Watch:**

Marcus shows signs of **Grid Hardening** risk—he may over-correct and create rigid systems that stifle the team. The antidote: involve employees in building the systems. Structure built together is structure that lives.

**The Real Question:**

Marcus asks "How do I build structure without killing the soul?" The deeper question is: "What am I afraid will happen if I'm not needed everywhere?" The business needs him to be the craft guardian, not the operational bottleneck.

**Recommended First Step:**

Start with **One Zone, One System**. Pick the most painful operational issue (likely inventory) and implement Two-Bin Kanban with Employee A as owner. Success here builds confidence for the next zone. Failure here teaches what not to do—cheaply.

**Integrity/Permeability Check:**

The systems must be permeable—revisitable, adjustable, improvable. Marcus should schedule a monthly "system review" where the team asks: "What's working? What's stupid? What should we change?" This prevents the bureaucracy trap.
