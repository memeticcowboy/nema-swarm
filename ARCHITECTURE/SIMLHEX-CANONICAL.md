# SIMLHEX CANONICAL — Routing Bias Layer

## Purpose

SIMLHEX maps SIML structural categories to elemental operator affinities. This mapping is **advisory only, non-executable, non-binding**. It biases attention — suggests which operator perspective might illuminate a given structure — without constraining action.

NEMA decides whether to act on the bias, hold it, or ask for human input. Automatic routing based on SIMLHEX is explicitly forbidden.

Each SIML Object has a default elemental affinity based on functional correspondence:

| **Narrative** | β (Wood) | Story, branching, temporal possibility |

| **Outcome** | λ (Fire) | Directional consequence, result of vectoring |

### Reading the table

The primary operator is a **starting bias** — the operator whose perspective is most likely to illuminate the given Object type. It is not the only perspective, and it is not always the right one.

**Example:** An Actor (λ bias) who is stuck may need σ (distinction) to clarify what they're actually doing, or ρ (resonance) to feel what drives them, before λ (direction) becomes useful.

## Bias Resolution Protocol

When Object and Relation biases conflict, NEMA uses this protocol:

1. **Note both biases:** "Object suggests λ, Relation suggests ρ"
2. **Hold without resolving:** The tension between biases is itself informative
3. **Suggest primary:** Lead with the stronger bias, note the secondary
4. **User decides:** Present both perspectives, let user choose

**Format for Discord EDS:**
```
SIMLHEX analysis:
  Object: Actor → λ (direction)
  Relation: Resonance → ρ (feeling)
  Bias tension: λ↔ρ (direction may need feeling-source)
  Suggestion: ρ first (source the direction), then λ (aim it)
  ε: Both biases are provisional. Hold lightly.
```

5. **Does not replace SIML.** Structure first, bias second.
6. **Does not touch governance.** NEME verb loop is outside SIMLHEX scope.
